

## خطة تحسين نظام المحادثات

### المشكلة الحالية
1. **هوية المستخدم الآخر غير واضحة**: يظهر "مستخدم" فقط بدون تحديد إن كان جمعية أو كادر
2. **لا يوجد رابط للملف الشخصي** في header المحادثة أو قائمة المحادثات
3. **التصميم أساسي** ويحتاج تحسين احترافي في كلا الصفحتين (TalentsMessages + Messages)

### التغييرات المطلوبة

#### 1. تحسين `useThreads` hook لجلب هوية كاملة
- جلب بيانات `user_roles` مع `organizations` للطرف الآخر
- تحديد نوع المستخدم: `talent` أو `org_member`
- إضافة حقول جديدة في `ThreadWithDetails`:
  - `user_type`: "talent" | "org" 
  - `org_name`: اسم الجمعية (إن وجد)
  - `org_logo`: شعار الجمعية
  - `org_slug`: رابط صفحة الجمعية
  - `profile_link`: رابط مباشر للملف (/talent/username أو /ngos/slug)

#### 2. تحسين واجهة TalentsMessages.tsx (بوابة الكادر)
- **قائمة المحادثات**:
  - إظهار شعار الجمعية أو صورة الكادر بشكل واضح
  - إظهار نوع المستخدم بـ Badge صغير ("جمعية" / "كادر")
  - اسم الجمعية/الكادر واضح وقابل للنقر
- **Header المحادثة**:
  - إظهار كامل هوية الطرف الآخر مع صورة كبيرة
  - زر "عرض الملف" ينقل لصفحة الجمعية أو الكادر
  - إظهار حالة "متصل" أو "آخر نشاط"
- **منطقة الرسائل**: تحسين الفقاعات والمسافات

#### 3. تحسين واجهة Messages.tsx (الصفحة العامة)
- نفس التحسينات أعلاه في عرض هوية المستخدم
- تحسين التصميم العام ليتوافق مع الصفحة الأخرى

#### 4. تحسين ChatWidget.tsx
- عند النقر على اسم الطرف الآخر في header → فتح ملفه الشخصي
- إظهار Badge نوع المستخدم

### الملفات المتأثرة

| ملف | تغيير |
|---|---|
| `src/hooks/useMessages.ts` | إضافة جلب user_roles + organizations لتحديد هوية المستخدم |
| `src/pages/talents/TalentsMessages.tsx` | إعادة بناء UI احترافي مع هوية كاملة |
| `src/pages/Messages.tsx` | تحسين عرض الهوية والتصميم |
| `src/components/chat/ChatWidget.tsx` | إضافة رابط الملف الشخصي |
